# IPTV 直播源本地文件上传指南

## 问题分析

GitHub Actions 工作流界面目前仅支持输入文件路径或 URL，无法直接上传本地文件。以下是几种可行的解决方案：

## 解决方案

### 方法 1：使用 URL（推荐）

将您的本地直播源文件上传到网络，然后在工作流中使用文件 URL。

#### 免费上传服务推荐
- GitHub Gist（支持文本文件）
- 奶牛快传
- 蓝奏云
- 123云盘
- 或您自己的服务器

#### 使用步骤
1. 上传本地文件到上述服务
2. 获取文件的公开访问 URL
3. 在工作流界面的「要检测的输入文件路径」字段中输入该 URL
4. 运行工作流

### 方法 2：推送到仓库后检测

将本地文件推送到仓库，然后在工作流中指定文件路径。

#### 使用步骤
1. 将本地直播源文件（如 `mylist.m3u`）复制到仓库根目录
2. 提交并推送到 GitHub
3. 在工作流界面输入文件路径（如 `mylist.m3u`）
4. 运行工作流

### 方法 3：使用特定目录上传

创建专门的上传目录，方便管理和检测。

#### 使用步骤
1. 在仓库中创建 `uploads` 目录
2. 将本地文件上传到该目录
3. 在工作流界面输入路径（如 `uploads/mylist.m3u`）
4. 运行工作流

### 方法 4：使用 GitHub API 自动化上传

对于高级用户，可以编写脚本使用 GitHub API 上传文件。

## 工作流改进

我们已经更新了 `validate_iptv.yml` 工作流，现在支持:

1. 输入本地文件路径
2. 输入网络文件 URL
3. 自动处理 M3U、TXT、JSON 格式

## 示例

### URL 示例
```
https://example.com/live.m3u
https://raw.githubusercontent.com/username/repo/master/live.txt
```

### 本地路径示例
```
mylive.m3u
uploads/channels.txt
```

## 注意事项

1. 文件大小建议不超过 10MB
2. 确保文件格式为 M3U、M3U8、TXT 或 JSON
3. URL 必须是公开可访问的
4. 本地路径是相对于仓库根目录的

## 验证结果

验证完成后，结果将保存在 `validator/results/` 目录中，包含：
- `*_valid.m3u` - 验证通过的 M3U 格式直播源
- `*_valid.txt` - 验证通过的 TXT 格式直播源

这些文件会自动提交并推送到仓库中供您下载使用。
